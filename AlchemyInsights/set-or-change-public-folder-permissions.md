# <a name="permissions-and-public-folders"></a>Tillatelser og fellesmapper

Du kan endre tillatelsene i fellesmappene ved hjelp av Outlook, administrasjonssenteret for Exchange (EAC) eller PowerShell:
  
- For Outlook-instruksjoner, [Klikk her](https://support.office.com/article/Set-or-change-permissions-for-a-public-folder-b2e0440c-7873-48ec-9ff2-b1a20b723005.aspx).
    
- For EAC, kan du se [denne artikkelen](https://technet.microsoft.com/library/jj651147%28v=exchg.150%29.aspx.aspx#Anchor_1) for instruksjoner. 
    
- For PowerShell, kan du se [denne artikkelen](https://technet.microsoft.com/library/bb124743%28v=exchg.160%29.aspx.aspx) for instruksjoner om hvordan du bruker tillegget PublicFolderClientPermission kommandlet. Hvis du trenger instruksjoner for å koble til Exchange PowerShell, kan du klikke [her](https://technet.microsoft.com/library/jj984289%28v=exchg.160%29.aspx.aspx).
    
Hvis **eksterne brukere ikke kan sende e-post til en e-post fellesmappe**, kan årsaken være at fellesmappen mangler tillatelser som kreves for ekstern e-postlevering. Du kan fikse dette ved hjelp av Outlook instruksjonene [her](https://technet.microsoft.com/library/aa997560%28v=exchg.150%29.aspx.aspx#Anchor_1), eller PowerShell instruksjonene [her](https://support.microsoft.com/help/2984402/-5.7.1-smtp-550-5.7.1-resolver.rst.authrequired-nondelivery-report-when-external-users-try-to-send-mail-to-mail-enabled-public-folders-in-office-365.aspx).
